另外一个大坑：

  • 有个大坑跟你分享下，你的blog开发新东西要部署上去，只要执行cap production deploy。capistrano就会自动跟去你的github把master这个分支部署上去。所以，要把改动的merge到master才会生效。

官方文档里有介绍可以直接推会支的方法，没试成功，有兴趣的童鞋可以去试试～

总结：

（在做这一课时）

  • 只字不差地阅读。
  • 教材上有没标出来的，也要尽量改得跟图片一样。
  • 请时刻清楚要操作的位置：服务器端？本地？home目录下？专案目录下？？
